# Verification Checklist

- [ ] Planned verification commands were run or listed as unavailable.
- [ ] Builder branch and commit range were checked.
- [ ] Each planned success criterion has a verification outcome.
- [ ] Failure behavior was checked when applicable.
- [ ] Named edge cases were checked or listed as unverified.
- [ ] Verifier made no product-code edits unless CEO authorized a fix branch.
- [ ] Unverified areas are listed.
- [ ] Residual risk is stated.
- [ ] Verification handoff lists checks, findings, unverified areas, and residual risk.
