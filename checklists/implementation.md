# Implementation Checklist

- [ ] Existing code patterns were inspected.
- [ ] Changes are limited to approved scope.
- [ ] Worktree path, branch, base commit, and head commit are recorded.
- [ ] Builder commit exists, or uncommitted diff command is recorded.
- [ ] Diff has no unrelated files or refactors.
- [ ] New abstractions are justified.
- [ ] User changes were not overwritten.
- [ ] Subagent outputs were integrated and checked.
- [ ] Known risks are included in the handoff.
- [ ] Builder handoff lists commit range, changed files, known risks, and requested next action.
