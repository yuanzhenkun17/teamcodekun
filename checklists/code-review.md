# Code Review Checklist

- [ ] Code satisfies the stated requirement.
- [ ] Code follows local style and patterns.
- [ ] Ambiguous names are flagged or none are present.
- [ ] Complex control-flow paths are traced or flagged.
- [ ] Error paths for named risks are handled or flagged.
- [ ] New or changed dependencies are listed with a reason, or no dependency changes were made.
- [ ] No secrets or sensitive data are exposed.
- [ ] Tests cover stated success criteria or gaps are listed.
- [ ] No unrelated refactor is included.
