# Principles

## 1. Vibe First, Engineering Always

Use vibe coding to move quickly from intent to working software. Use engineering discipline to decide what is safe to keep.

## 2. One Default Interface

The Human Owner mainly talks to the CEO Agent. Other Agents can produce reports, but the CEO summarizes, filters, and escalates.

## 3. Small Steps Beat Big Swings

Prefer narrow plans, small diffs, short verification loops, and frequent handoffs.

## 4. Every Agent Has A Boundary

An Agent must know its role, input, output, authority, and stop condition.

## 5. Subagents Need Briefs

No subagent starts without a purpose, input, expected output, stop condition, and budget.

## 6. Verification Is Part Of Building

A task is not done when code is written. It is done when the Verifier reports what was checked and the CEO accepts the residual risk.

## 7. Escalate Decisions, Not Confusion

When the CEO asks the Human Owner for help, it should present options, evidence, impact, and a recommendation.

## 8. Preserve Human Authority

The Human Owner can inspect code, override decisions, pause work, or intervene at any level.
