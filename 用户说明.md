# teamcodekun 用户说明

`teamcodekun` 用来让多个 AI 编程 session 按固定角色协作。

推荐配置：

```text
Codex：CEO
Claude Code：Builder
Codex：Verifier
```

## 1. 放到项目里

项目根目录结构：

```text
your-project/
  teamcodekun/
  your-code/
```

新项目可复制规则文件：

```bash
cp teamcodekun/project-template/AGENTS.md ./AGENTS.md
```

## 2. 初始化协作目录

在项目根目录运行：

```bash
./teamcodekun/scripts/init-run.sh current
```

会生成：

```text
.teamcodekun-runs/current/
  status.md
  ceo/
  builder/
  verifier/
  log/
.teamcodekun-run-path
```

三个 session 通过这些文件交接任务；`log/` 保存带时间戳的历史记录。

## 3. 三个窗口分别输入

CEO：

```text
请读取 ./teamcodekun/sessions/ceo.md，并开始任务。
```

Builder：

```text
请读取 ./teamcodekun/sessions/builder.md，并等待 CEO brief。
```

Verifier：

```text
请读取 ./teamcodekun/sessions/verifier.md，并等待 Builder handoff。
```

## 4. 协作流程

```text
你 -> CEO：提出需求
CEO -> Builder：写 plan 和 builder-brief
Builder -> Verifier：实现后写 handoff
Verifier -> CEO：验证后写 report
CEO -> 你：汇总结论
```

每次交接会同时写两份：

```text
最新状态：ceo/、builder/、verifier/ 里的固定文件
历史记录：log/YYYYMMDD-HHMMSS-角色-事件.md
```

默认规则：

- CEO 负责沟通、拆解、决策，不写产品代码；
- Builder 负责改代码；
- Verifier 负责检查和验证，默认不改代码。

## 5. 使用 Git worktree

可以让三个角色在不同 worktree 工作：

```text
your-project-ceo
your-project-builder
your-project-verifier
```

注意：

- 三个 session 必须使用同一个绝对路径 `RUN_DIR`；
- Builder 默认是唯一写代码角色；
- Verifier 只在 CEO 授权后才能修代码。

如果 `RUN_DIR` 用相对路径，不同 worktree 可能看不到彼此的交接文件。
